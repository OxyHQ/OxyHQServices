# Auth App (IdP)

The app that powers `auth.oxy.so`. It is the **OAuth 2.0 authorize/consent Identity Provider** for third-party "Sign in with Oxy", plus sign-in, sign-up, and password recovery for users arriving through an OAuth flow.

<Badge variant="success">App</Badge> <Badge>Vite + React Native Web</Badge>

## What it is

A small, focused app whose only job is third-party authorization. Third-party apps registered in the [Oxy Console](https://console.oxy.so) redirect users here with an OAuth authorize request and get back a short-lived, single-use authorization code.

It is **not** a relying party and **not** the session authority for first-party apps — official Oxy apps sign in in-app through `@oxyhq/services` and the device-first session model (see [device sessions](../../../docs/auth/device-session.md)); they never redirect here. Legacy browser-mediated identity flows (FedCM and related silent-restore machinery) were removed from the ecosystem.

Unlike the SDKs, this is a deployed app — you don't `bun add` it. You integrate by sending users to `/authorize` (see the [integration guide](../../../docs/auth/integration-guide.md)).

## Built on the shared SDK

The IdP renders `@oxyhq/services` components via React Native Web, so its sign-in and consent surfaces are pixel-identical to the in-app SDK ones:

```tsx
<OxyProvider baseURL={getApiBaseUrl()} clientId={OXY_CLIENT_ID} coldBoot={false}>
  {/* routes */}
</OxyProvider>
```

- `coldBoot={false}` — IdP mode. The provider supplies the auth UI context but performs no session restore of its own (the IdP is not an RP).
- The sign-in surface is the shared SDK one: Commons QR / deep-link first, password collapsed under "Sign in without the app".
- Consent renders `OxyConsentScreen` from `@oxyhq/services`, fed by the registered `Application` (name, logo, scopes, `privacyPolicyUrl`, `termsUrl`).

## Routes

| Route | Purpose |
|-------|---------|
| `/login` | Sign in (Commons QR / password) |
| `/signup` | Keyless password sign-up |
| `/authorize` | OAuth authorize + consent |
| `/recover` | Password recovery (request → code → reset) |
| `/auth/social/callback` | Social-login callback |
| `/settings/*` | **Permanent redirects to `accounts.oxy.so`** — the IdP does not own account management |

## Authorize flow (OAuth 2.0 + PKCE)

When a third-party app sends the user to `/authorize?client_id=…&redirect_uri=…&response_type=code&state=…&code_challenge=…&code_challenge_method=S256`:

1. The app resolves the registered Application via `GET /auth/oauth/client/:clientId` and validates the `redirect_uri` against `Application.redirectUris` (exact match).
2. If the user has no IdP session, the sign-in surface is shown first.
3. `GET /auth/oauth/consent` decides whether consent is required (`third_party` apps: yes, at least on first grant). If so, `OxyConsentScreen` lists the app identity, scopes, and legal links.
4. On approval, `POST /auth/oauth/authorize` mints a single-use code and the browser is redirected to `redirect_uri?code=…&state=…` — tokens are never in the URL.
5. The RP exchanges the code at `POST https://api.oxy.so/auth/oauth/token` with its PKCE `codeVerifier` (public clients) or `clientSecret` (confidential clients).

Users can review and revoke grants at any time in **Accounts → Connected apps** (`GET /auth/grants`, `DELETE /auth/grants/:applicationId`).

## Recovery

Multi-step:
1. `/recover` — enter email or username, app calls `POST /auth/recover/request`.
2. Enter the emailed code — `POST /auth/recover/verify`.
3. Choose a new password — `POST /auth/recover/reset`.

## Tests

`packages/auth` uses **Bun's native test runner** (`bun test`), not Jest — it is the one package in the monorepo with that setup. Run its own `bun run test` script.

## API surface

No JS API — this is a deployed app. Programmatic integration happens via:

- [`@oxyhq/core`](/developers/docs/core) — `generatePkcePair`, `generateOAuthState`, `buildOAuthAuthorizeUrl`
- [`@oxyhq/services`](/developers/docs/services) — `OxySignInButton` (starts the OAuth redirect automatically for `third_party` applications)
- The REST API's `/auth/oauth/*` endpoints — see the [API reference](/developers/docs/api)

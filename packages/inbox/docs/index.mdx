# Inbox

The Oxy email app. End-to-end encrypted at rest, offline-first via TanStack Query's persist-client + a Service Worker that precaches the app shell. Encryption keys are stored as non-extractable `CryptoKey` objects in IndexedDB so they can never leave the device.

## What it is

A user-facing email client built on the Oxy platform. The frontend is a Vite SPA; the backend lives in `@oxyhq/api` and handles per-account mailboxes, federation with external SMTP, and webhook ingest. Like MyAccount, Inbox is open source so the encryption + sync patterns are visible and auditable.

## Threat model

- **What the server sees:** ciphertext, message size, sender/recipient envelope (for routing), timing.
- **What the server cannot see:** message bodies, attachment contents, subject lines (encrypted with the body).
- **What a stolen device cannot do:** export the private key. The decryption `CryptoKey` is created with `extractable: false` and lives in IndexedDB only. Wiping IndexedDB destroys the key permanently.
- **Recovery:** the user's Oxy recovery phrase wraps the per-account encryption key. Losing the phrase and the device permanently destroys the mailbox.

## Architecture

```
   ┌────────────┐                       ┌────────────┐
   │ Inbox SPA  │  ────fetch────────▶  │  Oxy API   │
   │  (Vite)    │                       │  /inbox/*  │
   │            │  ◀──Server Push──────  │            │
   └────────────┘                       └────────────┘
        │                                     │
        ▼                                     ▼
  IndexedDB (keys,                      MongoDB / S3
  message cache,                        (ciphertext only)
  search index)
```

The SPA decrypts everything client-side. The server never sees plaintext.

## Encryption keys

- Generated on first sign-in via `crypto.subtle.generateKey({ name: 'AES-GCM', length: 256 }, false, ['encrypt', 'decrypt'])`.
- Stored in IndexedDB under a per-account key id; never exported to localStorage or sessionStorage.
- Wrapped by a key derived from the user's recovery phrase (PBKDF2 over the seed) so a fresh device can re-derive the same key after recovery-phrase login.
- Random IDs / nonces use `crypto.getRandomValues` — never `Math.random`.

## Offline & sync

- **TanStack Query persist-client** persists message list and thread queries to IndexedDB for instant first paint.
- **Service Worker manifest precaching** caches the app shell, fonts, and core JS chunks.
- **Mutations queue offline** — sending a message while offline writes a draft to IndexedDB and replays the network call on reconnect.
- **Background sync** uses the `BackgroundSync` API where available, falling back to in-page replay.

See the per-app `CLAUDE.md` for the exact cache strategy.

## Accessibility & typography

- Bloom typography pass — H1/H2/body all use Inter via `BloomThemeProvider`'s `Text.defaultProps`.
- Every interactive element has an `aria-label`; toolbars use `role="toolbar"`.
- Keyboard-first navigation: `j`/`k` move through threads, `e` archives, `?` opens the shortcut palette.

## API surface

Inbox is a deployed app — there is no JS API. To build a third-party mail client against the Oxy platform, talk to `/inbox/*` endpoints directly through the [REST API](/developers/docs/api). The client-side encryption logic is open source in `packages/inbox/src/crypto/` and can be lifted into another app, but the public-key choreography is specific to the Oxy account model.

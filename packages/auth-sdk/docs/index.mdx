# @oxyhq/auth

Headless web authentication for React. Provides `WebOxyProvider`, React Query-powered hooks, FedCM integration, and zero-config session sync over WebSocket. **No React Native or Expo dependencies.**

<Badge variant="success">SDK</Badge> <Badge>MIT</Badge>

## What it is

`@oxyhq/auth` is the web counterpart to `@oxyhq/services`. It owns the auth state, session lifecycle, and React Query cache for any Vite / Next.js / CRA app that talks to the Oxy API. Under the hood it delegates HTTP work to `OxyServices` from `@oxyhq/core`; on top, it exposes ergonomic hooks for the common UI patterns (read the current user, switch sessions, upload an avatar, manage devices).

## Install

```sh
bun add @oxyhq/auth @oxyhq/core
```

Peer dependencies: `react >= 18`, `@oxyhq/core`. Internal deps include `@tanstack/react-query`, `zustand`, `socket.io-client`, and `sonner`.

## Quick start

```tsx
import { WebOxyProvider, useAuth } from '@oxyhq/auth';

function App() {
  return (
    <WebOxyProvider baseURL="https://api.oxy.so">
      <Shell />
    </WebOxyProvider>
  );
}

function Shell() {
  const { user, isAuthenticated, signIn, signOut } = useAuth();
  if (!isAuthenticated) return <button onClick={() => signIn()}>Sign in</button>;
  return (
    <header>
      Welcome, {user?.name}
      <button onClick={() => signOut()}>Sign out</button>
    </header>
  );
}
```

The provider awaits `restored` (TanStack Query persist-client) before exposing the QueryClient — first paint serves cached data instead of a loading spinner.

## Core concepts

### Provider and context

`WebOxyProvider` owns:

- `OxyServices` (`@oxyhq/core`) — the HTTP client
- `QueryClient` (`@tanstack/react-query`) — caching and dedup
- A `localStorage`-backed persister so reloads keep cached `users`, `sessions`, `devices`, `privacy`, and mutation queues
- `AuthManager` — token refresh, multi-session switching, cross-domain SSO

Two hooks read from the provider:

- `useAuth()` — high-level surface: `user`, `isAuthenticated`, `signIn`, `signOut`
- `useWebOxy()` — full context: `oxyServices`, `sessions`, `switchSession`, `clearSessionState`, `accountStore`

### Query hooks

All query hooks are thin wrappers over `useQuery` with stable query keys so they participate in the persisted cache:

| Hook | What it fetches |
|------|-----------------|
| `useCurrentUser()` | `GET /users/me` (auto-refetched on focus) |
| `useUserById(id)` | `GET /users/:id` |
| `useUserByUsername(username)` | `GET /profiles/username/:username` |
| `usePrivacySettings(userId)` | privacy flags for a user |
| `useSessions()` | every active session for the current user |
| `useDeviceSessions(deviceId)` | sessions tied to a specific device |
| `useSecurityActivity()` / `useRecentSecurityActivity()` | recent security events |
| `useFollow(userId)` / `useFollowerCounts(userId)` | follow state + counts |

### Mutation hooks

Mutations carry stable `mutationKey`s and use `networkMode: 'offlineFirst'` so they queue while offline and replay on reconnect:

```tsx
const { mutateAsync: updateProfile } = useUpdateProfile();
await updateProfile({ name: 'Ada' });
// Optimistic update + targeted cache invalidation, all built in.
```

Available: `useUpdateProfile`, `useUploadAvatar`, `useUpdateAccountSettings`, `useUpdatePrivacySettings`, `useUploadFile`, `useSwitchSession`, `useLogoutSession`, `useLogoutAll`, `useUpdateDeviceName`, `useRemoveDevice`.

### Real-time session sync

`useSessionSocket()` opens a Socket.IO connection to the API and listens for session events. Zero config — every dependency is pulled from `WebOxyProvider`:

```tsx
function SessionGuard() {
  useSessionSocket(); // attaches once, returns nothing
  return null;
}
```

Optional callbacks for routing or analytics:

```tsx
useSessionSocket({
  onRemoteSignOut: () => router.push('/login'),
  onSessionRemoved: (sessionId) => track('session_removed', { sessionId }),
});
```

Only three event types ever trigger a local sign-out (`session_removed`, `device_removed`, `sessions_removed`); unknown events log a dev warning and do nothing.

## Common patterns

### Multi-session account switcher

```tsx
import { useWebOxy, useAccountStore } from '@oxyhq/auth';

function Switcher() {
  const { switchSession } = useWebOxy();
  const accounts = useAccountStore((s) => s.accounts);
  return accounts.map((a) => (
    <button key={a.sessionId} onClick={() => switchSession(a.sessionId)}>
      {a.handle}
    </button>
  ));
}
```

### Avatar upload with progress

```tsx
import { useUploadAvatar } from '@oxyhq/auth';

function AvatarPicker() {
  const { mutateAsync, isPending } = useUploadAvatar();
  return (
    <input
      type="file"
      accept="image/*"
      disabled={isPending}
      onChange={async (e) => {
        const file = e.target.files?.[0];
        if (!file) return;
        await mutateAsync(file);
      }}
    />
  );
}
```

### Cross-domain SSO via FedCM

```tsx
import { useAuth } from '@oxyhq/auth';

function SignInButton() {
  const { signIn } = useAuth();
  return (
    <button onClick={() => signIn({ method: 'fedcm' })}>
      Sign in with Oxy
    </button>
  );
}
```

`signIn` falls back to popup or redirect flows if FedCM is unavailable.

### Privacy-respecting error handlers

```ts
import { extractErrorMessage, isInvalidSessionError } from '@oxyhq/auth';

try {
  await oxy.users.updateProfile(input);
} catch (err) {
  if (isInvalidSessionError(err)) {
    router.push('/login');
    return;
  }
  toast.error(extractErrorMessage(err));
}
```

### Custom mutation with full pipeline

```ts
import { createGenericMutation } from '@oxyhq/auth';

const useArchiveNote = createGenericMutation({
  mutationKey: ['notes', 'archive'],
  mutationFn: (id: string) => oxy.notes.archive(id),
  invalidate: (queryClient, id) => {
    queryClient.invalidateQueries({ queryKey: ['notes'] });
    queryClient.invalidateQueries({ queryKey: ['notes', id] });
  },
});
```

## Offline persistence

The provider wires `@tanstack/react-query-persist-client` over `localStorage` (web) — `accounts`, `users`, `sessions`, `devices`, `privacy` queries persist for 30 days, mutations always persist. `useOnlineStatus()` exposes the online/offline state via `useSyncExternalStore` over `onlineManager` for offline banners.

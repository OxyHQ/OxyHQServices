# Hooks reference

`@oxyhq/auth` is hook-first. Below is the full catalogue grouped by purpose. Every hook is a thin wrapper over `useQuery` / `useMutation` with stable `queryKey`s / `mutationKey`s so they participate in the persisted cache and the offline queue.

## Auth surface

### `useAuth()`

The "I just want the basics" hook.

```tsx
const { user, isAuthenticated, signIn, signOut } = useAuth();
```

Returns the active user (or `null`), an `isAuthenticated` flag, and `signIn` / `signOut` callbacks that delegate to the configured `AuthManager`.

### `useWebOxy()`

Full context access — the OxyServices client, multi-session helpers, account store.

```tsx
const {
  oxyServices,
  sessions,
  activeSessionId,
  switchSession,
  clearSessionState,
} = useWebOxy();
```

Reach for this when you need the `OxyServices` instance directly (e.g. to call a mixin that doesn't have a hook wrapper yet) or when implementing a multi-session UI.

### `useWebSSO()`

Cross-domain SSO helper. Detects whether the current browser supports the SSO flow and exposes the imperative API.

### `isWebBrowser()`

Predicate for branching on the runtime — useful in code shared between SSR and SPA.

## Query hooks (reads)

| Hook | Endpoint | Notes |
|------|----------|-------|
| `useCurrentUser()` | `GET /users/me` | Refetches on window focus; persisted. |
| `useUserById(id)` | `GET /users/:id` | |
| `useUserByUsername(username)` | `GET /profiles/username/:username` | |
| `useUserProfile(id)` / `useUserProfiles(ids)` | Convenience wrappers | |
| `useUsersBySessions()` | Batched profile lookup for an array of sessions | |
| `usePrivacySettings(userId)` | `GET /privacy/:userId` | |
| `useSessions()` / `useSession(id)` | `GET /sessions` / `GET /sessions/:id` | |
| `useDeviceSessions(deviceId)` | Sessions for one device | |
| `useUserDevices(userId)` | Devices for one user | |
| `useSecurityInfo()` | `GET /security/info` | |
| `useSecurityActivity()` / `useRecentSecurityActivity()` | `GET /security/activity` | |

All queries follow the [`useCurrentUser` pattern](../core) — `queryFn` is pure, side effects live in event handlers or `useEffect`s on `query.data`.

## Mutation hooks (writes)

Every mutation uses `networkMode: 'offlineFirst'` and a stable `mutationKey` so it survives reloads and queues offline.

| Hook | Endpoint | Optimistic? |
|------|----------|-------------|
| `useUpdateProfile()` | `PUT /users/me` | Yes — optimistic merge + cache invalidation |
| `useUploadAvatar()` | `POST /files/upload-raw` → `PUT /users/me` | Optimistic url swap |
| `useUpdateAccountSettings()` | `PATCH /account/settings` | Yes |
| `useUpdatePrivacySettings()` | `PUT /users/:userId/privacy` | Yes — three-way merge in `onMutate` |
| `useUploadFile()` | `POST /files/upload-raw` | No (returns the new file) |
| `useSwitchSession()` | n/a (local) | Synchronous |
| `useLogoutSession(id)` | `DELETE /sessions/:id` | Optimistic removal from list |
| `useLogoutAll()` | `DELETE /sessions/all` | |
| `useUpdateDeviceName()` | `PATCH /devices/:id` | Yes |
| `useRemoveDevice()` | `DELETE /devices/:id` | Yes |

### Important: `useUpdatePrivacySettings`

Do **not** call `invalidateAccountQueries(queryClient)` in `onSuccess` — it defeats the optimistic merge. Use the three-way merge in `onMutate`:

```ts
onMutate: async (requested) => {
  await queryClient.cancelQueries({ queryKey: privacyKey });
  const previous = queryClient.getQueryData(privacyKey);
  queryClient.setQueryData(privacyKey, (incoming) => ({ ...previous, ...requested, ...incoming }));
  return { previous };
},
onError: (_err, _vars, ctx) => {
  if (ctx?.previous) queryClient.setQueryData(privacyKey, ctx.previous);
  queryClient.invalidateQueries({ queryKey: privacyKey });
},
```

## Custom mutations

For endpoints without a built-in hook, build one with the factories:

```ts
import { createGenericMutation } from '@oxyhq/auth';

const useArchiveNote = createGenericMutation({
  mutationKey: ['notes', 'archive'],
  mutationFn: (id: string) => oxy.notes.archive(id),
  invalidate: (queryClient, id) => {
    queryClient.invalidateQueries({ queryKey: ['notes'] });
    queryClient.invalidateQueries({ queryKey: ['notes', id] });
  },
});
```

`createProfileMutation` is similar but pre-wires the profile optimistic merge.

## Real-time hooks

### `useSessionSocket(options?)`

Zero-config Socket.IO connection that listens for `session_removed`, `device_removed`, `sessions_removed`. Optional callbacks:

```tsx
useSessionSocket({
  onRemoteSignOut: () => router.push('/login'),
  onSessionRemoved: (sessionId) => track('session_removed', { sessionId }),
});
```

Only the three whitelisted events ever trigger a local sign-out. Unknown events log a dev warning and do nothing.

## Asset hooks

`useAssets()` / `useAsset(id)` / `useUploadProgress()` / `useAssetLoading()` / `useAssetsByApp(appId)` etc. are stateful selectors over the zustand `assetStore`. `useFileDownloadUrl(fileId, variant)` returns the public URL for an asset.

## Follow hooks

`useFollow(userId)` and `useFollowerCounts(userId)` wrap the social graph endpoints with optimistic counts.

## Filtering helpers

`useFileFiltering()` exposes the view-mode / sort-by / sort-order trio used by the MyAccount file browser. `ViewMode | SortBy | SortOrder` types are exported for downstream UIs.

## Error helpers

```ts
import { extractErrorMessage, handleAuthError, isInvalidSessionError, isTimeoutOrNetworkError } from '@oxyhq/auth';
```

- `extractErrorMessage(err)` — returns a user-friendly string regardless of the source.
- `handleAuthError(err, { onInvalidSession })` — opinionated handler that calls back when the session is gone.
- `isInvalidSessionError`, `isTimeoutOrNetworkError` — narrow `unknown` to known error categories without `instanceof`.

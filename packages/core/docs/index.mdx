# @oxyhq/core

The platform-agnostic foundation of the Oxy SDK. **Zero React, zero React Native, zero Expo dependencies** — it runs identically in Node, Bun, Vite, Next.js, Cloudflare Workers, and React Native.

<Badge variant="success">SDK</Badge> <Badge>MIT</Badge>

## What it is

`@oxyhq/core` is the universal client, session, and crypto layer for the Oxy platform. The UI SDK (`@oxyhq/services`, for Expo/RN and web via React Native Web) builds on top of it. Use core directly when you need API access from a Node server, an Edge runtime, a service worker, or anywhere React is not in play.

The package ships two public entry points:

- `@oxyhq/core` — `OxyServices` client + mixins, `SessionClient` and the device-first cold boot, models, error types, platform helpers, `KeyManager` / `SignatureService` / `RecoveryPhraseService`, OAuth + PKCE helpers, and the shared color/theme/error utilities used across the Oxy ecosystem
- `@oxyhq/core/server` — Express-only helpers for request identity, rate limiting, CORS, SSRF-safe fetch, and constant-time secret comparison

All client/runtime symbols below are reachable from the package root. Server-only Express helpers live under `@oxyhq/core/server` so browser and React Native bundles never import Express.

## Install

```sh
bun add @oxyhq/core
```

The package is dual CJS + ESM and works in every JavaScript runtime that supports `fetch`.

## Quick start

```ts
import { OxyServices } from '@oxyhq/core';
import type { User } from '@oxyhq/core';

const oxy = new OxyServices({
  baseURL: 'https://api.oxy.so',
  cloudURL: 'https://cloud.oxy.so',
});

oxy.setTokens('ACCESS_TOKEN');
const me: User = await oxy.getCurrentUser();
```

## Core concepts

### OxyServices and mixins

`OxyServices` is composed from feature-specific mixins (`src/mixins/`); methods live directly on the instance:

| Mixin | Source | Representative methods |
|-------|--------|------------------------|
| Auth | `OxyServices.auth.ts` | `signIn`, `signUp`, `requestChallenge`, `verifyChallenge`, `getServiceToken` |
| Device session mint | `OxyServices.deviceBoot.ts` | `mintFromDeviceSecret` (zero-cookie `{deviceId, deviceSecret}` → access token) |
| User | `OxyServices.user.ts` | `getCurrentUser`, `getUserById`, `getProfileByUsername`, `updateProfile` |
| Assets | `OxyServices.assets.ts` | `assetUpload`, `getFileDownloadUrl`, `listUserFiles` |
| Privacy | `OxyServices.privacy.ts` | `getPrivacySettings`, `updatePrivacySettings` |
| Contacts | `OxyServices.contacts.ts` | `discoverContacts` (privacy-first, hashed identifiers) |
| Security | `OxyServices.security.ts` | sessions, devices, 2FA, security activity |
| Payment | `OxyServices.payment.ts` | payment methods, billing |
| Accounts | `OxyServices.accounts.ts` | account graph (orgs, projects, shared accounts), `switchToAccount` |
| Connected apps | `OxyServices.connectedApps.ts` | OAuth grants list / revoke |
| Identity | `OxyServices.identity.ts` | DID resolution, signed records, domain verification |

All methods return typed promises; errors throw `ApiError` subclasses you can narrow with helpers like `isInvalidSessionError`, `isRateLimitError`.

### Device-first sessions

The server-side `DeviceSession` is the single session authority: one document per device holding the signed-in accounts, the active account, and a revision counter. Core owns the client half of that contract:

- **`SessionClient`** (`src/session/`) — fetches `GET /session/device/state`, mutates via `POST /session/device/{add,switch,signout}`, and applies `session_state` socket pushes (room `device:<deviceId>`, token-free payload) so every Oxy app on the same device syncs instantly.
- **`runSessionColdBoot`** (`src/boot/sessionColdBoot.ts`) — deterministic, short-circuit cold boot. Restores silently when device state exists; otherwise resolves logged-out. It never redirects to a login page.
- **Zero-cookie transport** — every successful sign-in returns `deviceId` + a 256-bit `deviceSecret`, persisted first-party (localStorage per web origin; SecureStore on native). `mintFromDeviceSecret` (`OxyServices.deviceBoot.ts`) POSTs `{ deviceId, deviceSecret }` to `POST /session/device/token` (no bearer, no cookies) to mint/refresh a short access token and rotate the secret in-use. There is no cookie, no refresh-token family, and no `#oxy_boot` bootstrap hop. Full API and socket contract: [device sessions](../../../docs/auth/device-session.md).

React apps do not use `SessionClient` directly — `OxyProvider` from `@oxyhq/services` wires it up.

### OAuth + PKCE helpers (third party)

Third-party "Sign in with Oxy" is standard OAuth 2.0 Authorization Code + PKCE against `auth.oxy.so`:

```ts
import { generatePkcePair, generateOAuthState, buildOAuthAuthorizeUrl } from '@oxyhq/core';

const [pkce, state] = await Promise.all([generatePkcePair(), generateOAuthState()]);
const authorizeUrl = buildOAuthAuthorizeUrl({
  clientId: 'oxy_dk_…',
  redirectUri: 'https://merchant.example/auth/callback',
  codeChallenge: pkce.codeChallenge,
  state,
});
// Top-level navigate to authorizeUrl; exchange the returned code at POST /auth/oauth/token.
```

See the [integration guide](../../../docs/auth/integration-guide.md) for the full flow (SPA, confidential server, native custom scheme).

### Crypto and identity

`KeyManager` owns secp256k1 keypairs used for Oxy's recovery phrase login (all methods are static). It enforces atomic persistence, integrity probes, and refuses to clobber a healthy identity:

```ts
import { KeyManager, RecoveryPhraseService } from '@oxyhq/core';

if (!(await KeyManager.hasIdentity())) {
  const identity = await RecoveryPhraseService.generateIdentityWithRecovery();
  // Show `identity.phrase` to the user once, then forget it.
}
```

`KeyManager` throws `IdentityAlreadyExistsError` if you call `createIdentity` over an existing key — pass `{ overwrite: true }` to replace. See [KeyManager Safety](#keymanager-safety) below.

### User ids and profile handles

Oxy API payloads may expose either `id` or Mongo-style `_id`. Use the identity helpers before storing user payloads in app state:

```ts
import { getNormalizedUserId, normalizeUserIdentity } from '@oxyhq/core';

const userId = getNormalizedUserId(user);
const normalized = normalizeUserIdentity(user);
```

For display and profile routing, use `getNormalizedUserHandle()` instead of rebuilding handles in each app. It removes a leading `@`, keeps an existing `user@instance` handle intact, and appends `instance` / `federation.domain` only for federated users:

```ts
import { getNormalizedUserHandle } from '@oxyhq/core';

getNormalizedUserHandle({ username: 'alice' });
getNormalizedUserHandle({ username: 'alice', isFederated: true, instance: 'example.social' });
```

`User.name.displayName` is **optional** on SDK user/profile payloads — federated
or unresolved actors routinely omit it. Render it when present; when absent,
fall back to the normalized handle (`displayName ?? handle`). Do not rebuild
names from `name.first`, `name.last`, `name.full`, or `username`.

### Service tokens

For backend-to-backend calls inside the Oxy ecosystem, register an `Application` with an `ApplicationCredential` of `type: 'service'` and use the OAuth2 client-credentials flow. The credential's `publicKey` (`oxy_dk_…`) is the client id; the secret is shown once at create/rotate time:

```ts
const oxy = new OxyServices({ baseURL: 'https://api.oxy.so' });
oxy.configureServiceAuth('oxy_dk_…', 'secret…');

// Auto-cached, auto-refreshed service JWT.
const token = await oxy.getServiceToken();

// Or delegate as a specific user with one call:
const result = await oxy.makeServiceRequest('POST', '/some/endpoint', body, userId);
```

The API recognises service tokens (`type: 'service'` in the JWT payload) statelessly — no session lookup, no DB round-trip. Wrap protected endpoints with `oxy.serviceAuth()` middleware. See [Service Tokens](../../../docs/SERVICE_TOKENS.md).

### HTTP service

`OxyServices` shares a single `HttpService` instance (`oxy.httpService`) that handles:

- Bearer token attachment + refresh
- Request deduplication
- Per-prefix in-memory cache (`clearCacheByPrefix` clears specific paths)
- Retry with exponential backoff for 5xx / network errors
- Request queuing during token refresh

## Common patterns

### Sign in with a recovery phrase

```ts
import { OxyServices, RecoveryPhraseService, SignatureService } from '@oxyhq/core';

const oxy = new OxyServices({ baseURL: 'https://api.oxy.so' });

const publicKey = await RecoveryPhraseService.restoreFromPhrase(mnemonic);

const { challenge } = await oxy.requestChallenge(publicKey);
const signed = await SignatureService.signChallenge(challenge);
const session = await oxy.verifyChallenge(
  signed.publicKey,
  challenge,
  signed.challenge, // the signature over the auth message
  signed.timestamp,
);
// verifyChallenge plants the access token internally — no manual setTokens needed.
```

### Update profile and bust the cache

```ts
await oxy.updateProfile({ name: { first: 'Ada' } });
// updateProfile already clears `GET:/users/me`, `GET:/session/user/`,
// and `GET:/profiles/username/` — but if you have custom cached
// reads, call clearCacheByPrefix yourself.
oxy.httpService.clearCacheByPrefix('GET:/my-custom-endpoint/');
```

### Privacy-first contact discovery

```ts
const sha256Hex = async (input: string) => {
  const buf = new TextEncoder().encode(input.trim().toLowerCase());
  const digest = await crypto.subtle.digest('SHA-256', buf);
  return Array.from(new Uint8Array(digest))
    .map((b) => b.toString(16).padStart(2, '0'))
    .join('');
};

const emailHashes = await Promise.all(emails.map(sha256Hex));
const matches = await oxy.discoverContacts(emailHashes, []);
// Only matching Oxy users come back; raw emails never leave the device.
```

## KeyManager Safety

`KeyManager` enforces strict invariants:

- `createIdentity` / `importKeyPair` throw `IdentityAlreadyExistsError` unless `{ overwrite: true }` is passed.
- Writes use a backup-first, sign/verify-probe-after atomic strategy; failures roll back to the backup.
- `hasIdentity()` requires both keys present, well-formed, and matching — not just key existence.
- `verifyIdentityIntegrity()` performs a real sign/verify probe rather than byte parsing.
- `restoreIdentityFromBackup()` refuses to clobber a healthy primary or switch users.
- Strict hex/length/range validation on every private/public key.

Treat the private key as load-bearing — never log it, never send it over the wire.

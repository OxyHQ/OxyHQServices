# Mixins

`OxyServices` is composed from a dozen feature-specific mixins so the public surface stays organised and tree-shakers can drop unused namespaces. Every mixin lives in `packages/core/src/mixins/OxyServices.<feature>.ts` and contributes a single namespace under the client.

## How composition works

```ts
// packages/core/src/mixins/index.ts
export function composeOxyServices(Base) {
  return composeMixins(Base,
    OxyServicesAuth,
    OxyServicesUser,
    OxyServicesAssets,
    /* … one per feature … */
  );
}

// packages/core/src/OxyServices.ts
export class OxyServices extends composeOxyServices(OxyServicesBase) { /* … */ }
```

Each mixin extends the base, adds new methods, and returns the augmented class. The end result is a single `OxyServices` constructor whose prototype carries the union of every feature.

## Namespaces

### `oxy.auth`

Login, signup, refresh, TOTP enrollment, service tokens, recovery.

```ts
const session = await oxy.auth.login({ identifier, password });
await oxy.auth.refresh();
await oxy.auth.logout();

await oxy.auth.totpEnrollStart(sessionId);
await oxy.auth.totpEnrollVerify(code);

oxy.configureServiceAuth('oxy_dk_…', 'secret…');
const token = await oxy.getServiceToken();
```

### `oxy.users`

Profile read/write, follow, search.

```ts
const me = await oxy.users.getMe();
await oxy.users.updateProfile({ name: 'Ada' });
await oxy.users.follow(targetUserId);
const results = await oxy.users.search(query);
```

### `oxy.files`

Upload, download URL builder, list, metadata.

```ts
await oxy.files.upload(file);
const url = oxy.files.getDownloadUrl(fileId, 'thumb');
const list = await oxy.files.list(userId);
```

### `oxy.privacy`

Get and update privacy flags.

```ts
const settings = await oxy.privacy.getSettings(userId);
await oxy.privacy.updateSettings(userId, { profileVisibility: 'public' });
```

`PrivacySettings` is a strongly typed interface — no `Record<string, any>`.

### `oxy.contacts`

Privacy-first contact discovery — hash on the client, server matches without ever seeing raw PII.

```ts
const sha256Hex = async (input: string) => {
  const buf = new TextEncoder().encode(input.trim().toLowerCase());
  const digest = await crypto.subtle.digest('SHA-256', buf);
  return Array.from(new Uint8Array(digest)).map((b) => b.toString(16).padStart(2, '0')).join('');
};

const emailHashes = await Promise.all(emails.map(sha256Hex));
const matches = await oxy.contacts.discoverContacts(emailHashes, []);
```

Rate limited to 200 hashes per request, 5 requests/min/user.

### `oxy.security`

Sessions, devices, recent activity.

```ts
const sessions = await oxy.security.listSessions();
await oxy.security.removeSession(sessionId);
const activity = await oxy.security.getActivity({ limit: 50, cursor });
```

### `oxy.devices`

Device management — names, sessions per device, removal.

```ts
const devices = await oxy.devices.list();
await oxy.devices.updateName(deviceId, 'Ada's MacBook');
await oxy.devices.remove(deviceId);
```

### `oxy.payment`

Payment methods + billing operations.

```ts
const methods = await oxy.payment.listMethods();
await oxy.payment.addMethod({ /* provider-specific token */ });
```

### `oxy.managedAccounts`

Sub-accounts (e.g. an organisation owner managing per-team accounts).

```ts
const child = await oxy.managedAccounts.create({ name: 'Marketing', parentId });
await oxy.managedAccounts.actAs(childId);
```

### Other namespaces

`oxy.reputation`, `oxy.location`, `oxy.language`, `oxy.topics`, `oxy.identity`, `oxy.appData`, `oxy.civic`, `oxy.nodes`, `oxy.links`, `oxy.connectedApps`, `oxy.analytics`, `oxy.features`, `oxy.utility` round out the surface (the developer-app mixin was replaced by the Applications model — see `oxy.managedAccounts` above; FedCM was removed ecosystem-wide — see [device sessions](../../../docs/auth/device-session.md)). Each is documented in detail under the auto-generated **API Reference** in the sidebar.

## Adding a mixin

For internal use only — third-party consumers should not extend `OxyServices`. The pattern:

```ts
// packages/core/src/mixins/OxyServices.notes.ts
import type { Constructor } from './mixinHelpers';
import type { OxyServicesBase } from '../OxyServices.base';

export function OxyServicesNotes<TBase extends Constructor<OxyServicesBase>>(Base: TBase) {
  return class extends Base {
    notes = {
      list: () => this.request('GET', '/notes'),
      create: (input: NoteInput) => this.request('POST', '/notes', input),
    };
  };
}
```

Then add it to the `composeOxyServices` call. New mixins inherit the same auth, cache, retry, and dedup behaviour as everything else.
